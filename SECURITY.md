# Security Policy

## Supported Versions

Paper2Repo is currently pre-1.0. Security fixes are handled on the latest `main` branch and the latest published release.

| Version | Supported |
| --- | --- |
| `v0.1.x` | Yes |
| Older versions | No |

## Reporting a Vulnerability

Please do not open a public issue with exploit details, private data, or credentials.

Use GitHub private vulnerability reporting or a repository security advisory when available. If that option is unavailable, open a minimal public issue asking for a secure contact path and do not include sensitive details.

Useful reports include:

- A short description of the risk.
- Reproduction steps or affected endpoints.
- Whether the issue affects local files, uploaded PDFs, generated reports, API keys, or the SQLite database.
- Any safe proof of concept that does not expose secrets or third-party data.

The project aims to acknowledge valid reports promptly and document the fix in a release note when appropriate.
